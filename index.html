ArchDaily styles guidelines and Alfred usage

CSS guides

Our stylesheets are writed mainly based on this css guidelines, but there are six core principes thtat you don't have to ignore if you are styling for ArchDaily

  • 1. One file for each component
  • 2. Name classes using BEM
  • 3. Never style html tags and avoid style IDs. Use only classes
  • 4. Avoid indentate css selectors
  • 5. Write clear code, someone else will have to deal with it
  • 6. Name always from the general to the particular

1. One file for each component

The title says everything. If you have to style a new commponent create a new file. It helps to make easyer to debug and the edit. If this component is going to be deprecated in the future you won't have problems cleaning the styles

3. Never style html tags and avoid style IDs. Use only classes

That's it! The experience tell us that it allways could cause style colisions, a nigthmare debuging, fixing and styling other components. Don't do it, we have made too much mistakes with it and we have to learn. If you use good BEM you don't need to style tags or IDs

4. Avoid indentate css selectors

This could make conflicts of inheritance with the styles and make slower the process that the machine make of the styles. If you are using SCSS probably could think that it is awesome, but is not. In fact you are under risk of to get terrible compilated selectors. Once again, if you use good BEM you don't need to style with nested selectors.

5. Write clear code, someone else will have to deal with it

This could be relative, but here are a few practices that are helping us to make it clear.

6. Name always from the general to the particular

This is a rule for class names and variables names mainly. The reason is to make easyer the reading of them.
